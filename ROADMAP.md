# План доработок (кратко)

Цель: зафиксировать потенциальные улучшения и их приоритет. Сейчас выполнения не требуется.

## P1 — Важное (делать при появлении боли)
1. Фоновый воркер для `recommendation_jobs`.
   - Причина: задачи выполняются только при открытой странице.
   - Эффект: стабильная генерация рекомендаций без участия клиента.
   - Оценка: 1–2 дня (cron‑endpoint + обработчик + мониторинг).
2. Rate‑limit для `/api/feedback/submit` и `/api/feedback/token`.
   - Причина: публичная форма может быть целью спама.
   - Эффект: защита от злоупотреблений.
   - Оценка: 0.5–1 день (Upstash/Edge limiter + honeypot).

## P2 — Полезно (по мере роста)
1. Централизованное логирование ошибок (Sentry/аналог).
   - Эффект: быстрее ловить ошибки продакшна.
   - Оценка: 0.5–1 день.
2. Материализованные представления/кеш для аналитики.
   - Эффект: стабильная скорость при больших объёмах данных.
   - Оценка: 2–4 дня (SQL + cron refresh).

## P3 — Опционально (по желанию)
1. Service layer (services/repositories).
   - Эффект: улучшение тестируемости и структуры.
   - Оценка: 2–5 дней, зависит от объёма.
2. OpenAPI‑документация.
   - Эффект: удобство для интеграций и команды.
   - Оценка: 1–2 дня.
