# Mbooking: план внедрения подписки через ЮKassa

Документ описывает внедрение подписки 500 ₽ / 30 дней через ЮKassa как основной или параллельный платежный канал.

---

## 1) Что берем из ЮKassa для вашего кейса

Для SaaS-подписки полезно:
- API-интеграция для сайта;
- оплата по ссылке как fallback без сложной фронтенд-логики;
- поддержка автоплатежей для будущего автопродления;
- несколько способов оплаты (карты, СБП и др.).

Базовый целевой флоу:
- пользователь оплачивает 500 ₽;
- webhook подтверждает успех;
- подписка активируется на 30 дней.

---

## 2) Тариф и правила доступа

- Тариф: `500 ₽ / 30 дней`.
- До оплаты: закрыты `lessons` и `recommendations` (paywall).
- После оплаты: полный premium-доступ.
- По истечении периода: возврат на ограниченный режим.

---

## 3) Архитектура backend

Использовать общий billing-модуль и провайдерный слой:
- `BillingProvider` (интерфейс);
- `YooKassaProvider` (реализация).

Преимущество: в любой момент можно держать параллельно Яндекс Пэй/ЮKassa.

---

## 4) Prisma-модели

Использовать единые таблицы:
- `OwnerSubscription`
- `PaymentTransaction`

Ключевое:
- `provider = yookassa`
- `providerPaymentId` уникальный
- idempotency key сохраняется в транзакции

---

## 5) Backend API для ЮKassa

- `POST /v1/billing/yookassa/checkout`
  - создает платеж;
  - возвращает confirmation URL.

- `POST /v1/billing/yookassa/webhook` (public)
  - принимает уведомления;
  - обрабатывает `payment.succeeded` как источник истины.

- `GET /v1/billing/subscription`
  - отдает текущий статус подписки.

---

## 6) Логика активации подписки

При `payment.succeeded`:
- в транзакции фиксируется `succeeded`;
- подписка:
  - `now + 30 days`, если не активна;
  - `currentPeriodEnd + 30 days`, если уже активна.

Важно:
- подтверждение только через webhook;
- по return-url не активировать доступ.

---

## 7) Frontend интеграция

Добавить/использовать:
- `app/billing/page.tsx`
- `app/paywall/page.tsx`
- `app/billing/success/page.tsx`
- `app/billing/cancel/page.tsx`

API:
- `src/api/billing-yookassa.api.ts`
- `src/hooks/billing.hooks.ts`

Гейтинг:
- `app/recommendations/*` и `app/lessons/*` через SSR проверку подписки.

---

## 8) Комплаенс и операционные нюансы (важно)

Если работаете как самозанятый:
- учитывать лимиты режима НПД;
- учитывать правила по формированию чеков и требованиям к деятельности;
- отслеживать ограничения по обороту и видам деятельности.

Проверки до запуска:
- юридическая модель (самозанятый/ИП/ООО);
- нужные способы оплаты;
- фактические комиссии и SLA зачисления.

---

## 9) Пошаговый план внедрения

1. Подключить ЮKassa, получить shop credentials (test/prod).
2. Реализовать `YooKassaProvider` и checkout endpoint.
3. Реализовать webhook endpoint и идемпотентный обработчик.
4. Добавить `SubscriptionAccessService` и premium-guards.
5. Подключить frontend billing/paywall flow.
6. Прогнать тесты:
   - success/cancel;
   - duplicated webhook;
   - delayed webhook;
   - amount mismatch.
7. Включить прод-мониторинг платежных событий.

---

## 10) Когда выбирать ЮKassa как основной канал

- нужен проверенный сценарий для подписок и автосписаний;
- нужен быстрый старт с оплатой через ссылку как запасной режим;
- важно иметь несколько платежных методов "из коробки".

---

## 11) KPI после запуска

- conversion to paid;
- success rate платежей;
- доля активных подписок D30/D60;
- churn после первого периода;
- вовлеченность в premium (`lessons`, `recommendations`).
